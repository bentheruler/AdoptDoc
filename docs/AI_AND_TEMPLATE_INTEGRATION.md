#  AI & Template Integration: Developer's Guide

Hey team!.I've officially set up the brain of our system. This guide will help you understand how the AI generation and document rendering works so you can start "tapping" into these APIs from the frontend.

---

## The "Plug-and-Play" Architecture



### 1. The AI Abstraction Layer (with Failover)
We are using **Google Gemini** as our primary AI engine because it's fast and smart. However, because we want 99.9% uptime, I've implemented a **Failover Manager**.
- **Primary**: Gemini (`GeminiService.js`)
- **Secondary**: OpenAI (`OpenAIService.js`)
- **How it works**: If Gemini takes more than 5 seconds or fails, the system automatically switches to OpenAI. This happens in the background without the user ever knowing.

### 2. Prompt Engineering & JSON Schema
The AI is strictly instructed to return **Valid JSON**. 
- For **CVs**, we use the [JSON Resume Schema](https://jsonresume.org/schema).
- For **Cover Letters** and **Proposals**, we use custom structured JSON.
This makes it super easy for us to save the data to MongoDB and render it later.

### 3. The Rendering Engine (Standard HTML/CSS)
Instead of using complex templating engines, I used **Standard JavaScript Template Literals**. 
- Located in: `server/templates/`
- Why? This means anyone on the team can edit the HTML/CSS of a theme just by editing a standard JS string. No new tools to learn!

##  How to "Tap" into the API

Here are the endpoints -ready:

### 1. Generate Content
**Endpoint**: `POST /api/ai/generate`  
**Body**:
```json
{
  "docType": "resume",
  "userData": { "name": "John Doe", "skills": ["JS", "React"] }
}
```
**Response**: Returns the structured JSON content generated by the AI.

### 2. Get Themes
**Endpoint**: `GET /api/templates/themes`  
**Response**: Returns a list of all available themes for CVs, Cover Letters, and Proposals.

### 3. Live Preview
**Endpoint**: `GET /api/documents/:id/preview?theme=classic`  
**Usage**: Put this URL inside an `<iframe>` on the frontend. It will render the document HTML live!

## 🧪 How to Test

I've added a simple test script in the root directory called `test-api.js`. This is the easiest way to verify everything is working.

### 1. Start the Server
In your first terminal:
```bash
node server.js
```

### 2. Run the Test Script
In your second terminal:
```bash
node test-api.js
```

This script will:
1.  Check if the **Themes API** is returning our document layouts.
2.  Check if the **Live Preview** is generating HTML.
3.  Check if the **AI Generation** is working (make sure you have your Gemini or OpenAI API key in `server/.env`).

### 📝 Note on Environment Variables
Make sure you have your `GEMINI_API_KEY` and `OPENAI_API_KEY` in your `.env` file inside the `server/` folder. I've left placeholders there for you.
